# 관리자 홈 데일리 브리프·스케줄 체크리스트 디자인

## 목표

관리자 홈 상단의 긴 데일리 브리프를 짧고 밀도 높은 2분할 카드로 바꾼다. 왼쪽은 현재 브리프를 유지하고, 오른쪽은 오늘 일정을 직접 완료 처리할 수 있는 체크리스트를 제공한다.

## 레이아웃

- 하나의 짙은 초록색 카드 안을 데스크톱에서 50:50으로 나눈다.
- 왼쪽에는 날짜, 현재 시각, 인사말, 대기 건수, 다음 일정을 표시한다.
- 오른쪽에는 `오늘 스케줄` 제목, 완료 수, 일정 체크리스트, 캘린더 이동 링크를 표시한다.
- 두 영역 사이에는 낮은 대비의 세로 구분선을 둔다.
- 카드의 전체 높이는 기존보다 줄이고, 체크리스트가 많으면 오른쪽 목록 내부에서 스크롤한다.
- 모바일에서는 왼쪽 브리프 위, 오른쪽 체크리스트 아래의 단일 열로 배치한다.
- 기존 상단 `오늘의 명언` 카드는 제거한다.

## 데이터와 정렬

- 기존 `/api/dashboard` 응답의 `todayTasks`를 양쪽 영역이 공유한다.
- 미완료 일정을 먼저 표시하고, 각 그룹 안에서는 시작 시간순으로 정렬한다.
- 일정 시간, 제목, 완료 상태를 표시한다.
- 일정이 없으면 `오늘 등록된 일정이 없습니다.`라는 빈 상태와 캘린더 이동 링크를 표시한다.
- 로딩 중에는 체크리스트 스켈레톤을, 조회 실패 시에는 재시도 안내를 표시한다.

## 체크 동작

- 체크박스를 누르면 화면에 즉시 완료/미완료 상태를 반영한다.
- `PATCH /api/calendar`에 `{ id, completed }`를 전송해 `calendar_tasks`에 저장한다.
- 저장 중인 항목은 중복 클릭을 막고 진행 상태를 표시한다.
- 저장 성공 후 완료 수, 남은 할 일 수, 진행률, 다음 일정 표시를 갱신한다.
- 저장 실패 시 이전 상태로 되돌리고 해당 목록 안에 오류 메시지를 표시한다.

## 컴포넌트 범위

- `TodayAlertBanner`가 조회 상태와 일정 상태, 완료 저장을 관리한다.
- 브리프와 체크리스트를 같은 컴포넌트 안의 독립된 두 패널로 구성한다.
- 관리자 홈에서는 `DailyQuoteWidget` 렌더링을 제거한다. 명언 데이터 파일 자체는 다른 사용 가능성을 위해 삭제하지 않는다.
- 캘린더 페이지와 API의 기존 동작은 변경하지 않는다.

## 접근성

- 각 일정은 실제 `button` 또는 체크박스 의미가 전달되는 컨트롤을 사용한다.
- 완료 상태는 `aria-pressed` 또는 동등한 체크 상태로 전달한다.
- 저장 오류와 로딩 상태는 보조 기술에 안내한다.
- 키보드만으로도 각 일정 완료 상태를 변경할 수 있어야 한다.

## 완료 기준

1. 데스크톱에서 상단 카드가 정확히 두 영역으로 보인다.
2. 왼쪽에는 기존 브리프 정보가 유지된다.
3. 오른쪽에서 오늘 일정을 직접 완료·해제할 수 있다.
4. 변경 사항이 캘린더 데이터에 저장되고 실패 시 원복된다.
5. 일정이 많아도 카드 전체가 과도하게 길어지지 않는다.
6. 모바일에서는 두 영역이 읽기 좋은 단일 열로 전환된다.
7. 타입 검사와 프로덕션 빌드가 통과한다.
