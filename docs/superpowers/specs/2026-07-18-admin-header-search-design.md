# 관리자 헤더 검색 및 AI Assistant 명칭 개편 설계

## 목적

관리자 홈과 기능 홈의 헤더를 실제 업무 탐색의 시작점으로 만든다. 관리자 홈에는 고객·프로젝트·기능 통합검색과 알림·빠른 실행·프로필을 제공하고, 기존 `개별 기능` 영역은 `AI Assistant`로 명칭을 바꾸며 기능 검색을 헤더로 이동한다.

## 확정 범위

### 관리자 홈

- 헤더 제목을 `포토클리닉 AI 비서`로 변경한다.
- 현재 compact 제목 크기에서 약 20% 축소한다.
- 헤더 오른쪽에 통합검색, 알림, 빠른 실행, 프로필을 배치한다.
- 본문에 있는 `빠른 실행` 섹션은 삭제한다.

### AI Assistant 홈

- `/admin/tools` 헤더 제목을 `AI Assistant`로 변경한다.
- 현재 compact 제목 크기에서 약 20% 축소한다.
- 기존 본문 기능 검색바를 헤더 오른쪽으로 이동한다.
- 사이드바의 `개별 기능`은 `AI Assistant`, `개별 기능 홈`은 `AI Assistant 홈`으로 변경한다.
- 기능 카드와 실제 기능 라우트는 이름이나 경로를 변경하지 않는다.

## 구조

기존 `AdminHeader`를 경로에 따라 동작하는 공통 헤더로 확장한다.

```text
[로고 + 제목]                         [검색] [알림] [빠른 실행] [프로필]
```

- 로고와 제목은 헤더 왼쪽에 고정한다.
- 검색부터 시작되는 명령 그룹은 남은 공간의 오른쪽 끝에 정렬한다.
- AI Assistant 홈에서도 기능 검색창을 동일하게 오른쪽 끝에 정렬한다.
- 관리자 홈에서는 검색이 `global` 모드로 동작한다.
- AI Assistant 홈에서는 검색이 `tools` 모드로 동작한다.
- 그 외 관리자 세부 페이지는 기존 로고·제목 중심 헤더를 유지한다.
- 검색과 팝오버 상태는 헤더 전용 클라이언트 컴포넌트에서 관리한다.
- 고객·프로젝트 서버 검색은 별도 API로 격리하고, 기능 목록은 `lib/toolNav.ts`를 단일 소스로 사용한다.

## 관리자 홈 통합검색

### 데이터 소스

- 고객: 기존 고객 테이블에서 병원명·담당자 기준 검색
- 프로젝트: 활성 및 최근 워크플로우에서 고객명·프로젝트명 기준 검색
- 기능: `ALL_TOOLS`의 제목·설명·영문 메타 기준 검색

### API

`GET /api/admin/search?q=<검색어>`를 추가한다.

응답은 하위 호환성에 영향을 주지 않는 신규 형태로 제공한다.

```typescript
{
  ok: true,
  customers: SearchResult[],
  projects: SearchResult[],
  tools: SearchResult[]
}
```

서버 API는 고객·프로젝트 결과 수를 제한하고 전화번호·이메일 같은 개인정보를 반환하지 않는다. 기능 검색은 API 장애와 무관하게 클라이언트에서 계속 동작한다.

### 상호작용

- 두 글자 이상 입력하면 고객·프로젝트 검색을 실행한다.
- 입력 중 요청은 debounce하고 이전 요청은 취소한다.
- 결과는 `고객`, `프로젝트`, `기능` 그룹으로 나눈다.
- 고객을 선택하면 고객 상세 화면, 프로젝트를 선택하면 해당 워크플로우 문맥, 기능을 선택하면 기능 페이지로 이동한다.
- 키보드 위·아래 방향키와 Enter를 지원한다.
- 바깥 클릭과 Escape로 닫는다.

## AI Assistant 기능검색

- 기존 본문 검색 입력을 제거한다.
- 헤더 검색어를 URL의 `q` query parameter에 반영한다.
- `/admin/tools` 페이지는 `q`를 읽어 `ALL_TOOLS`를 서버에서 필터링한다.
- 검색 대상은 기능명, 설명, 영문 메타다.
- 결과가 없으면 빈 상태 안내와 `검색 초기화` 동작을 제공한다.
- 고객 연결용 query parameter인 `clientId`, `projectId`, `workflowRunId`, `stepKey`는 검색 중에도 보존한다.

## 알림

- 단순 장식 버튼으로 두지 않는다.
- 기존 `olivia_insights`, `olivia_actions`, `agent_approvals`, `olivia_events`에서 긴급·승인 대기·최근 고객 반응을 조회한다.
- 알림 버튼을 누르면 최근 항목을 compact 팝오버로 보여준다.
- 확인할 항목이 있으면 오렌지 점을 표시한다.
- 항목 선택 시 올리비아 비서 영역 또는 관련 고객·프로젝트로 이동한다.
- 데이터 일부 조회가 실패하면 조회 가능한 항목만 표시한다.

## 빠른 실행

헤더 빠른 실행 메뉴에는 다음 세 항목만 둔다.

- 일정 추가 → `/calendar`
- 메모 작성 → `/memo`
- 고객 등록 → `/clients`

기존 관리자 홈 본문의 빠른 실행 섹션은 삭제한다. 헤더 버튼은 딥그린 계열로 표현하고 오렌지는 알림 상태와 핵심 강조에만 사용한다.

## 프로필

- 포토클리닉 로고 마크와 `정연호 대표`를 표시한다.
- 이번 범위에서는 계정 설정 편집 기능을 추가하지 않는다.
- 모바일에서는 이름을 숨기고 아이콘만 유지한다.

## 반응형 디자인

- 데스크톱에서는 모든 헤더 요소를 한 줄로 배치하고 명령 그룹을 오른쪽 정렬한다.
- 좁은 화면에서는 검색창을 검색 아이콘으로 축소한다.
- 검색 아이콘을 누르면 헤더 아래에 전체 너비 검색 입력을 펼친다.
- 알림·빠른 실행·프로필은 아이콘 형태로 유지한다.
- 팝오버는 화면 가장자리를 넘지 않으며 모바일에서 거의 전체 너비를 사용한다.

## 접근성과 오류 처리

- 모든 아이콘 버튼에 명확한 `aria-label`을 제공한다.
- 검색 결과 목록은 combobox/listbox 의미를 제공한다.
- 로딩, 결과 없음, 서버 오류 상태를 구분한다.
- 검색 API 오류 시 고객·프로젝트 오류 안내와 함께 기능 검색 결과는 유지한다.
- 팝오버가 열리면 Escape로 닫을 수 있고 포커스가 검색 입력 또는 트리거로 돌아간다.

## 완료 기준

1. 관리자 홈 제목이 `포토클리닉 AI 비서`로 표시되고 기존보다 약 20% 작다.
2. 관리자 홈에서 고객·프로젝트·기능을 통합검색할 수 있다.
3. 알림 팝오버에서 긴급·승인·고객 반응을 확인할 수 있다.
4. 헤더 빠른 실행에서 일정·메모·고객 등록으로 이동할 수 있다.
5. 관리자 홈 본문의 빠른 실행 섹션이 제거된다.
6. `개별 기능` 명칭이 관리자 구조에서 `AI Assistant`로 변경된다.
7. AI Assistant 홈 검색이 헤더로 이동하고 실제 기능 카드를 필터링한다.
8. 검색 중 기존 CRM 연결 query parameter가 유지된다.
9. 모바일 헤더와 팝오버가 화면 밖으로 넘치지 않는다.
10. 단위 테스트, TypeScript 검사, 프로덕션 빌드가 통과한다.
11. 관리자 홈과 AI Assistant 홈 모두 검색창부터 시작되는 명령 그룹이 헤더 오른쪽 끝에 정렬된다.
