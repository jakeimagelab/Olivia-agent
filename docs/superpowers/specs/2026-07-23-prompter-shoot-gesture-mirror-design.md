# 프롬프터 촬영 체크·제스처·태블릿 미러 설계

## 범위

1. 프로젝트 씬 목록에서 촬영 완료 여부를 체크하고 저장한다.
2. 문단 또는 슬라이드마다 대본과 분리된 제스처를 편집하고 실행화면에 표시한다.
3. PC가 실행 호스트가 되고 QR로 접속한 기기는 리모트 모드와 PC 화면 미러링 모드를 전환할 수 있다.
4. 휴대폰은 기본적으로 압축 리모컨 UI를, 태블릿은 기본적으로 PC 미러링 UI를 추천하되 자동 판별 결과를 사용자가 언제든 바꿀 수 있다.

## 데이터 모델

- `prompter_scripts.is_shot boolean not null default false`
- `prompter_scripts.gesture_map jsonb not null default '[]'`
- `gesture_map`은 텍스트 모드에서는 문단 인덱스, 슬라이드 모드에서는 슬라이드 인덱스와 대응한다.
- 문단/슬라이드를 이동하거나 삭제하면 화자와 제스처 배열도 함께 이동·삭제한다.
- 신규 컬럼이 아직 없는 DB에서도 기존 씬 조회·저장이 실패하지 않도록 API 폴백을 둔다.

## 촬영 완료 UI

- 씬 목록 왼쪽에 체크박스를 둔다.
- 체크한 씬은 위치를 유지하고 제목과 날짜를 흐리게 표시한다.
- 체크 변경은 씬 전체 저장과 분리된 PATCH 요청으로 즉시 저장한다.

## 제스처 편집과 실행

- 텍스트 모드에 `제스처 표현` 편집 모드를 제공한다.
- 활성화 시 각 카드에 `대본`과 `제스처` 입력 영역을 분리해 보여준다.
- 다중 화자 모드와 함께 사용할 수 있다.
- 슬라이드 모드에서는 각 슬라이드 카드에 제스처 입력란을 항상 제공한다.
- 실행화면에는 해당 문장 옆에 강조된 작은 글씨로 `(제스처)`를 표시한다.
- 제스처가 비어 있으면 실행 레이아웃에 빈 공간을 만들지 않는다.

## 리모트·미러링 모드와 동기화

- PC가 세션 코드와 Realtime 채널을 만들고 상태의 최종 권한을 가진다.
- QR 화면 상단에 `리모트 모드`와 `미러링 모드` 전환 버튼을 항상 제공한다.
- 첫 접속에서는 터치 지원, 사용자 에이전트와 화면 크기를 함께 사용해 휴대폰에는 리모트 모드, 태블릿에는 미러링 모드를 추천한다.
- 사용자가 선택한 모드는 브라우저 로컬 저장소에 보관하고 다음 QR 접속에서도 우선 적용한다.
- 리모트 모드는 현재 휴대폰용 작은 미리보기와 압축 컨트롤을 그대로 유지한다.
- 미러링 모드는 PC와 동일한 공용 실행화면 컴포넌트를 사용한다. 타이머, 진행률, 대본, 가이드라인, 상단 버튼, 하단 설정 바를 모두 포함한다.
- 미러링 화면은 PC 뷰포트 크기로 렌더링한 전체 실행화면을 태블릿 영역에 비율만 맞춰 축소한다. 화면 상단의 작은 모드 전환 버튼만 PC 화면에 없는 예외 UI다.
- 해상도의 픽셀 일치보다 조작 연동 지연과 상태 일치를 우선한다.
- PC와 원격 기기 모두 동일한 명령 형식을 사용하고, 재생/정지 같은 상태는 `toggle`이 아니라 확정값을 전송한다.
- 모든 명령과 상태 프레임에는 세션 내 단조 증가 순번과 송신 기기 ID를 넣는다. 수신 측은 이미 처리했거나 늦게 도착한 순번을 무시한다.
- 원격 기기는 조작 결과를 화면에 즉시 낙관적으로 반영한 뒤 PC로 전송한다. PC는 적용한 확정 상태와 순번을 즉시 다시 방송한다.
- 스크롤 조작을 시작한 기기는 짧은 제어권을 얻는다. 제어권을 가진 동안 반대편의 오래된 스크롤 프레임이 현재 손가락 조작을 되돌리지 못한다.
- 스크롤은 애니메이션 프레임 기준으로 제한해 전송하고, 손을 떼거나 관성 스크롤이 멈추면 마지막 위치를 반드시 확정 전송한다.
- 같은 네트워크에서는 반대편 화면 반영 100ms 이내, 일반 인터넷에서는 200ms 안쪽을 목표로 하되 네트워크 상태에 따라 달라질 수 있다.
- 재접속 시 콘텐츠와 전체 상태를 다시 요청해 복구한다.

## 전체화면

- 브라우저 기본 Fullscreen API는 운영체제의 스와이프 종료 제스처를 차단할 수 없으므로 미러링 모드에서 사용하지 않는다.
- `전체화면` 버튼은 화면을 `position: fixed; inset: 0`으로 고정하는 앱 내부 몰입 모드를 켜고 끈다.
- 페이지의 당겨서 새로고침과 루트 오버스크롤을 차단하되, 대본 스크롤과 설정 조작은 유지한다.
- 앱 내부 몰입 모드는 오직 화면의 전체화면 버튼으로만 종료한다.
- 브라우저 주소창까지 없는 진짜 독립 전체화면이 필요한 경우에만 홈 화면 설치형 PWA를 후속 범위로 둔다.

## 컨트롤

- 재생/정지, 처음, 이전/다음 문단 또는 슬라이드
- 속도, 글자 크기, 문단 간격, 줄 간격
- 좌우/상하 반전, 가로/세로 정렬
- 글자색, 배경색
- 가이드라인 켜기, 위치, 문단 강조
- 버튼 전용 앱 내부 전체화면

## 검증

- 타입 검사, 단위 테스트, 프로덕션 빌드
- 휴대폰 크기에서 리모트 모드가 기존 압축 UI를 유지하는지 확인
- 태블릿 크기에서 미러링 모드가 자동 추천되는지 확인
- 양쪽 크기에서 수동 모드 전환과 선택 저장이 동작하는지 확인
- PC/원격 기기 브라우저 두 세션으로 재생·스크롤·문단 이동·모든 설정을 양방향으로 조작하고 상태가 일치하는지 확인
- 원격 스크롤 중 PC의 이전 프레임이 원격 화면을 되돌리지 않는지 확인
- 명령 순번이 뒤바뀌어 도착해도 최신 상태가 유지되는지 단위 테스트
- 앱 내부 전체화면이 스와이프로 종료되지 않고 버튼으로만 종료되는지 태블릿 실기기 확인
- 씬 체크와 제스처 저장 API의 구버전 DB 폴백 확인
- 검증 후 요청에 따라 Git 커밋·푸시 및 Vercel Production 배포
