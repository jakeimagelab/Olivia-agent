# 프롬프터 촬영 체크·제스처·태블릿 미러 설계

## 범위

1. 프로젝트 씬 목록에서 촬영 완료 여부를 체크하고 저장한다.
2. 문단 또는 슬라이드마다 대본과 분리된 제스처를 편집하고 실행화면에 표시한다.
3. PC가 실행 호스트가 되고 QR로 접속한 태블릿은 PC 프롬프터 화면을 실시간 미러링한다.
4. 휴대폰은 압축 리모컨 UI를 유지하되 재생 버튼을 확대하고 PC의 실행 설정을 모두 제공한다.

## 데이터 모델

- `prompter_scripts.is_shot boolean not null default false`
- `prompter_scripts.gesture_map jsonb not null default '[]'`
- `gesture_map`은 텍스트 모드에서는 문단 인덱스, 슬라이드 모드에서는 슬라이드 인덱스와 대응한다.
- 문단/슬라이드를 이동하거나 삭제하면 화자와 제스처 배열도 함께 이동·삭제한다.
- 신규 컬럼이 아직 없는 DB에서도 기존 씬 조회·저장이 실패하지 않도록 API 폴백을 둔다.

## 촬영 완료 UI

- 씬 목록 왼쪽에 체크박스를 둔다.
- 체크한 씬은 위치를 유지하고 제목과 날짜를 흐리게 표시한다.
- 체크 변경은 씬 전체 저장과 분리된 PATCH 요청으로 즉시 저장한다.

## 제스처 편집과 실행

- 텍스트 모드에 `제스처 표현` 편집 모드를 제공한다.
- 활성화 시 각 카드에 `대본`과 `제스처` 입력 영역을 분리해 보여준다.
- 다중 화자 모드와 함께 사용할 수 있다.
- 슬라이드 모드에서는 각 슬라이드 카드에 제스처 입력란을 항상 제공한다.
- 실행화면에는 해당 문장 옆에 강조된 작은 글씨로 `(제스처)`를 표시한다.
- 제스처가 비어 있으면 실행 레이아웃에 빈 공간을 만들지 않는다.

## PC·태블릿 동기화

- PC가 세션 코드와 Realtime 채널을 만들고 상태의 최종 권한을 가진다.
- QR 경로로 접속한 기기는 터치 지원 및 화면의 짧은 변 600px을 기준으로 휴대폰/태블릿을 구분한다.
- 태블릿은 PC 뷰포트 크기와 프롬프터 상태를 받아 동일한 화면 비율로 축소 렌더링한다.
- 태블릿 하단에는 항상 컨트롤을 표시하고 재생 버튼을 가장 크게 둔다.
- 휴대폰은 작은 미리보기와 압축 컨트롤을 유지한다.
- PC는 재생 중 스크롤 프레임을 제한된 주기로 방송하고 태블릿은 이를 부드럽게 보정한다.
- 태블릿 명령은 PC에 전송되고, PC가 확정한 상태가 다시 모든 미러에 방송된다.
- 재접속 시 콘텐츠와 전체 상태를 다시 요청해 복구한다.

## 컨트롤

- 재생/정지, 처음, 이전/다음 문단 또는 슬라이드
- 속도, 글자 크기, 문단 간격, 줄 간격
- 좌우/상하 반전, 가로/세로 정렬
- 글자색, 배경색
- 가이드라인 켜기, 위치, 문단 강조
- 태블릿 로컬 전체화면

## 검증

- 타입 검사, 단위 테스트, 프로덕션 빌드
- PC/태블릿 크기의 브라우저 두 세션으로 연결·명령·스크롤·가이드 동기화 확인
- 씬 체크와 제스처 저장 API의 구버전 DB 폴백 확인
- 검증 후 요청에 따라 Git 커밋·푸시 및 Vercel Production 배포
