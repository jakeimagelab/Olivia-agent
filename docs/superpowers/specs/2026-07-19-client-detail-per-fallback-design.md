# 고객 상세 PER 컬럼 폴백 설계

작성일: 2026-07-19

## 확인된 문제

운영 `GET /api/clients`는 `반포리움성형외과` 고객 행을 정상 반환한다. 그러나 `GET /api/clients/{id}`는 404를 반환한다.

운영 DB를 읽기 전용으로 확인한 결과 `clients` 테이블에는 다음 PER 확장 컬럼이 아직 없다.

- `total_paid_amount`
- `available_points`
- `total_earned_points`
- `reward_tier`

고객 상세 API는 기본 고객 컬럼과 위 확장 컬럼을 하나의 SELECT에 포함한다. PostgREST는 컬럼 하나라도 없으면 전체 SELECT를 실패시키며, 현재 API는 모든 조회 오류를 `고객을 찾을 수 없습니다`로 바꾸고 있어 실제 원인을 숨긴다.

`original_photos_link`, `retouched_photos_link` 컬럼과 해당 고객 행은 운영 DB에 정상 존재한다.

## 확정 변경

### 단계별 고객 조회

1. PER 확장 컬럼을 포함한 전체 고객 SELECT를 먼저 실행한다.
2. 오류가 누락 컬럼 오류일 때만 기본 고객 및 사진 링크 컬럼으로 다시 조회한다.
3. 폴백 조회 결과에는 다음 기본값을 보완한다.

```ts
{
  total_paid_amount: 0,
  available_points: 0,
  total_earned_points: 0,
  reward_tier: "standard",
}
```

권한 오류, 네트워크 오류 등 다른 오류는 폴백으로 숨기지 않는다.

### 오류 응답

- 실제 행이 없을 때만 404 `고객을 찾을 수 없습니다`를 반환한다.
- DB 조회 오류는 500과 실제 안전한 오류 메시지를 반환한다.
- 프론트는 API 오류 메시지를 고객 없음과 구분해 표시한다.

### PER 마이그레이션

`supabase/per-schema.sql`은 그대로 유지한다. 운영 DB에 이 마이그레이션을 적용하면 전체 SELECT가 사용되고, 폴백 기본값 대신 실제 PER 데이터가 표시된다.

이번 코드 수정은 마이그레이션 적용 전에도 고객 상세가 열리게 만드는 호환 계층이다. 기존 고객을 삭제하거나 다시 생성하지 않는다.

## 테스트

- 전체 고객 SELECT 성공 시 실제 PER 값을 유지한다.
- PER 컬럼 누락 오류일 때 기본 컬럼으로 재조회한다.
- 폴백 결과에 PER 기본값을 채운다.
- 실제 고객 미존재는 404로 구분한다.
- 다른 DB 오류는 고객 없음으로 바뀌지 않는다.
- 전체 단위 테스트, TypeScript 검사, 프로덕션 빌드가 통과한다.

## 완료 기준

- `반포리움성형외과` 상세 화면이 기존 고객 ID로 열린다.
- 운영 DB에 PER 컬럼이 없어도 기본 고객정보와 워크플로우를 볼 수 있다.
- PER 마이그레이션 적용 후 코드를 다시 수정하지 않아도 실제 포인트가 표시된다.
- 기존 고객 데이터 삭제나 재등록이 필요 없다.
