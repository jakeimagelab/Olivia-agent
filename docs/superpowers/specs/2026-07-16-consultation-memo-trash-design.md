# Olivia 상담 메모 및 공통 휴지통 개편 설계

## 목표

Olivia의 `/memo`를 상담 기록 작성, 필기, 음성 녹음, AI 분석을 한곳에서 처리하는 단순한 상담 작업실로 개편한다. 사용자가 직접 삭제하는 모든 주요 데이터는 즉시 영구 삭제하지 않고 공통 휴지통으로 이동하며, 원래 기능과 복원 가능 상태를 명확히 표시한다.

`/consultation`과 고객 등록 모달은 기존 고객 등록 흐름을 유지한다. 이번 개편의 주 화면은 `/memo`다.

## 범위

### 포함

- 이전 상담 기록의 휴지통 이동
- 상담 메모 템플릿 6종
- 터치, Apple Pencil, 마우스를 지원하는 드로잉 팔레트
- 필기·그림의 AI 텍스트 정리 및 이미지 정돈
- 상담 음성 1건 녹음, Supabase Storage 저장, 전사 및 요약
- 모든 사용자 직접 삭제 기능을 위한 공통 휴지통
- 휴지통 복원, 영구 삭제, 30일 만료 정보

### 제외

- 상담 한 건 안의 다중 페이지 노트북
- 자동 만료 파일, 캐시, 재업로드·동기화 과정의 내부 정리 데이터
- 삭제 전 데이터를 소급해 휴지통에 복원하는 작업
- 휴지통 항목의 복잡한 폴더 분류 체계

## 상담 메모 모델

상담 기록 한 건은 템플릿 한 장, 캔버스 한 개, 음성 파일 최대 한 개를 가진다. 텍스트와 구조화된 템플릿 값은 JSON으로 저장하고, 캔버스와 음성 같은 대용량 바이너리는 Supabase Storage에 저장한다.

기존 `consultation_memos`에는 다음 정보를 추가한다.

- `title`: 기록 제목
- `template_type`: `text`, `cornell`, `todo`, `blank`, `grid`, `conti`
- `template_data`: 템플릿별 텍스트, 체크 항목, 콘티 행·열과 캡션
- `canvas_path`: 원본 필기 이미지 Storage 경로
- `ai_image_path`: AI가 정돈한 이미지 Storage 경로
- `audio_path`: 음성 Storage 경로
- `audio_duration_seconds`: 녹음 길이
- `transcript`: 수정 가능한 전사문
- `audio_summary`: 음성 중심 요약
- `updated_at`: 마지막 수정 시각

기존 `raw_memo`, `summary`, `extracted_data`, `recommended_package`, `next_action`은 호환성을 위해 유지한다.

## 템플릿

- 기본 텍스트: 제목과 자유 텍스트
- 코넬형: 핵심 키워드, 본문, 하단 요약
- 할 일: 완료 상태를 가진 체크 항목과 항목 추가
- 백지: 자유 필기·스케치
- 모눈종이: 격자 배경 위 필기·스케치
- 콘티: `2×2`, `2×3`, `3×3` 프리셋과 사용자 지정 행·열. 각 칸은 16:9 프레임과 짧은 캡션을 제공한다.

템플릿을 바꿀 때 기존 내용이 손실될 수 있으면 확인 절차를 거친다.

## 드로잉 팔레트

기존 `DrawingCanvas`의 공용 엔진을 확장해 다른 콘티 화면과 동작을 공유한다.

- 연필: 낮은 불투명도와 미세 질감
- 볼펜: 일정하고 선명한 선
- 만년필: 입력 속도와 PointerEvent 압력에 따른 굵기 변화
- 형광펜: 반투명 합성, 전용 색상
- 펜 굵기와 지우개 크기 슬라이더
- 공통 색상 팔레트
- 부분 지우개, 전체 지우기
- 실행 취소, 다시 실행
- 선, 화살표, 사각형, 원

입력은 Pointer Events와 pointer capture를 사용한다. 히스토리는 스트로크·도형 단위로 관리하고 메모리 사용량에 상한을 둔다. 캔버스 자동 저장은 스트로크 종료 후 디바운스하며 저장 상태를 화면에 표시한다.

## AI 변환

### 텍스트로 정리

현재 템플릿의 텍스트, 캔버스 이미지, 음성 전사문을 함께 분석한다. 결과는 요약, 결정 사항, 요청 사항, 할 일, 촬영 아이디어를 포함하는 구조화된 미리보기로 제공한다. 사용자가 적용해야 원문이나 분석 결과에 반영한다.

### 이미지로 정리

캔버스 원본의 배치와 의도를 참고해 정돈된 디지털 노트 또는 콘티 이미지를 생성한다. 생성 결과는 원본을 덮어쓰지 않고 별도 미리보기와 Storage 파일로 저장한다. 실패해도 원본 캔버스는 보존된다.

API 키와 모델 오류는 사용자에게 재시도 가능한 상태로 표시한다. 빈 캔버스에서는 변환을 요청하지 않는다.

## 음성 메모

상담 기록 한 건에 음성 파일 한 개를 연결한다.

- 녹음, 일시정지, 이어 녹음, 종료
- 로컬 미리 듣기 후 저장
- Supabase Storage 업로드
- 전사문 생성과 사용자 수정
- 상담 핵심 내용, 결정 사항, 후속 작업 요약
- 기존 텍스트 AI 분석에 전사문 포함

새 녹음으로 교체할 때 이전 파일은 저장 완료 후 제거한다. 업로드나 전사에 실패하면 로컬 미리보기와 재시도 상태를 유지한다.

## 공통 휴지통

공통 `trash_items` 테이블은 다음 메타데이터를 가진다.

- `id`
- `source_type`: 원래 기능의 안정적인 키
- `source_label`: 화면에 표시할 기능명
- `source_table`
- `source_id`
- `title`과 `preview`
- `payload`: 삭제 직전 레코드 JSON 스냅샷
- `asset_paths`: 함께 복원하거나 영구 삭제할 Storage 경로
- `deleted_at`, `expires_at`
- `deleted_by`: 인증 정보가 있으면 사용자 식별자, 없으면 `system-user`

`/trash`는 분류 폴더 없이 최신순 목록을 보여주되 각 행에 원래 기능 배지, 제목, 미리보기, 삭제 시각, 남은 보관 기간을 표시한다. 기능명 검색·필터는 제공하지만 항목을 별도 분류함으로 이동하지 않는다.

### 삭제

사용자가 삭제를 확인하면 서버가 원본 레코드를 읽고 휴지통 스냅샷을 먼저 생성한다. 휴지통 저장이 성공한 경우에만 원본을 삭제한다. 관련 Storage 파일은 30일 동안 유지한다.

### 복원

복원 시 원래 테이블과 식별자를 우선 사용한다. 동일 식별자가 이미 존재하면 덮어쓰지 않고 충돌 상태를 반환한다. 부모 데이터가 필요한 항목은 부모가 존재할 때만 복원하며, 불가능한 이유를 화면에 표시한다. 복원 성공 후 휴지통 항목을 제거한다.

### 영구 삭제와 만료

영구 삭제는 별도 확인을 거치며 되돌릴 수 없다는 문구를 표시한다. 관련 Storage 파일을 먼저 삭제하고 휴지통 레코드를 제거한다. 자동 만료는 30일이 지난 항목을 대상으로 하는 보호된 정리 API로 제공하며, 예약 실행 연결 여부와 무관하게 수동 실행으로 검증 가능해야 한다.

### 연결 대상

현재 확인된 사용자 직접 삭제 기능인 상담 메모, 캘린더 일정, 고객, 메일, 저장 콘티, 콘티 드로잉, 영상 콘티, 리워드 상품, 트렌드 경쟁사, Olivia 대화 기록을 공통 휴지통 흐름에 연결한다. 갤러리 재처리처럼 기존 값을 교체하기 위한 내부 삭제는 제외한다.

## 화면 구조

`/memo`는 다음 순서로 구성한다.

1. 새 상담 메모와 이전 기록 진입점
2. 템플릿 선택
3. 제목·템플릿 편집 영역
4. 드로잉 팔레트와 캔버스
5. 음성 녹음·재생·전사 영역
6. AI 텍스트·이미지 변환과 결과 비교
7. 저장 상태와 상담 AI 분석 결과

이전 기록 목록은 불러오기와 삭제 동작을 제공한다. 삭제 후 목록에서 즉시 사라지고 실행 취소 안내를 통해 휴지통으로 이동할 수 있다.

`/trash`는 기능 배지가 포함된 단일 목록, 검색·기능 필터, 복원, 영구 삭제를 제공한다. 모바일에서는 작업 버튼이 충분한 터치 영역을 유지하며 목록 카드로 전환한다.

## 오류 및 보안

- 모든 쓰기 API는 입력 크기와 허용된 source type을 검증한다.
- 클라이언트가 전달한 임의의 테이블명을 직접 쿼리에 사용하지 않는다.
- Storage 경로는 서버가 생성하고 허용된 버킷·prefix만 삭제한다.
- 서비스 역할 키와 OpenAI 키는 서버에서만 사용한다.
- 음성·캔버스의 공개 URL을 영구 저장하지 않고 Storage 경로와 만료되는 signed URL을 사용한다.
- 삭제·복원·영구 삭제는 부분 성공을 명시적으로 처리하고 원본 유실보다 중복 보존을 우선한다.

## 검증 기준

- 각 템플릿 생성, 저장, 재로딩이 동일한 내용을 복원한다.
- 콘티 `2×3`은 한 줄에 두 칸씩 세 줄로 생성된다.
- 터치·마우스 입력, 펜 스타일, 굵기·색상, 지우개, 실행 취소·다시 실행, 전체 지우기, 도형이 동작한다.
- AI 변환은 미리보기 승인 전 원본을 변경하지 않는다.
- 음성 업로드 실패 후 재시도가 가능하고 성공 시 재생·전사·요약이 재로딩된다.
- 각 사용자 삭제 API는 휴지통 스냅샷 성공 뒤 원본을 제거한다.
- 휴지통에 원래 기능명이 표시되며 복원 후 원래 화면에서 다시 조회된다.
- 영구 삭제는 DB 스냅샷과 관련 Storage 파일을 제거한다.
- 동일 ID 충돌과 부모 누락 시 복원은 안전하게 중단된다.
- 기존 메모 AI 분석, 고객 등록, 캘린더·견적·콘티 이동 흐름이 회귀하지 않는다.
- TypeScript 검사와 프로덕션 빌드가 통과한다.
- Chromium에서 데스크톱과 모바일 핵심 사용자 흐름을 검증한다.
