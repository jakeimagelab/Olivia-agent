# 셀렉매칭 — 파일 순서 검토 기능 설계

## 배경

`app/(photo-studio)/select-match/page.tsx`(사진작업실 > 셀렉매칭)는 이미 상단 탭으로 두 기능을
전환하는 구조다: `feature` state가 `"raw_match"`(🎯 셀렉 & RAW 매칭) / `"find_move"`(📋 파일명으로
찾아 이동) 두 값을 갖고, 각자 독립된 `Step`/`fmStep` 상태와 화면 흐름을 가진다.

촬영 후 폴더를 넘겨받았을 때, 카메라 파일명 넘버링(예: `DSC03532.CR3`)이 중간에 끊겨 있으면
파일이 누락된 것 — 지금은 이를 확인할 방법이 없어 수작업으로 훑어봐야 한다. 이를 자동으로
검사하는 세 번째 탭을 추가한다.

## 범위

- 세 번째 `feature` 값(`"seq_check"`)으로 기존 탭 전환 UI에 자연스럽게 통합한다.
- 폴더는 하나만 선택하고, **하위 폴더는 뒤지지 않는다** (직속 파일만 검사).
- 검사 대상 확장자는 기존 페이지의 `RAW_EXTS`/`JPG_EXTS` 상수를 그대로 재사용한다(영상 확장자는
  이 페이지의 기존 범위에 포함되지 않으므로 추가하지 않는다 — 필요해지면 후속 작업).
- 카메라 접두사(DSC, IMG 등)나 자릿수는 신경 쓰지 않고, **폴더 전체를 하나의 숫자 순서**로 본다
  (사용자 확인 완료 — 접두사가 섞여도 한 시퀀스로 취급).
- RAW/JPG가 같은 번호를 공유해도(예: `DSC03532.CR3` + `DSC03532.JPG`) 문제 없이 동작해야 한다 —
  번호의 존재 여부만 확인하고 중복은 무시한다.

## 아키텍처

### 1. 탭 전환

`FeatureTabs` 컴포넌트에 세 번째 버튼 `🔢 파일 순서 검토` 추가, `feature` 타입을
`"raw_match" | "find_move" | "seq_check"`로 확장.

### 2. 번호 추출 규칙

파일명(확장자 제외)의 **맨 끝에서부터 연속된 숫자**를 정규식으로 추출한다: `/(\d+)$/`.
예: `DSC03532` → `3532`, `IMG_20260708_0091` → `91`(맨 끝 숫자만, 날짜 부분은 무시).
숫자를 전혀 못 찾으면 "인식 불가" 목록에 넣고 순서 검사 대상에서 제외한다.

이 규칙은 접두사·자릿수에 관계없이 카메라 기종에 상관없이 동작하는 가장 단순하고 견고한
방식이다 (해당 페이지의 다른 곳에서도 파일명 패턴에 특별한 가정을 두지 않는 것과 일관됨).

### 3. 검사 로직 (신규 순수 함수, 페이지 파일 내부에 추가)

```typescript
interface SequenceCheckResult {
  totalFiles: number;
  recognizedNumbers: number[];      // 중복 제거 + 정렬됨
  unrecognizedFiles: string[];      // 번호를 못 찾은 파일명
  missingRanges: { start: number; end: number }[]; // end - start + 1 = 누락 개수
  min: number | null;
  max: number | null;
}

function checkFileSequence(fileNames: string[]): SequenceCheckResult
```

- 정규식으로 각 파일명에서 숫자 추출 → `Set<number>`로 중복 제거 → 정렬.
- 최소~최대 사이를 순회하며 `Set`에 없는 연속 구간을 `missingRanges`로 묶는다.
- 파일이 1개 이하이거나 인식된 번호가 없으면 "검사 불가" 상태로 별도 안내.

### 4. 화면 흐름 (기존 페이지의 `Step`과 별개의 로컬 상태로 관리)

```
scStep: "idle" | "scanning" | "result"
```

- **idle**: 폴더 선택 버튼 하나 (`showDirectoryPicker`, 기존 페이지의 `hasFS` 체크 재사용) +
  "폴더 지정 후 파일 순서를 검토합니다" 안내문.
- **scanning**: 폴더 직속 엔트리를 순회하며 확장자 필터링 + 파일명 수집 (즉시 끝날 정도로
  가벼운 작업이라 진행률 UI는 간단한 스피너 텍스트로 충분).
- **result**:
  - 상단 요약 카드: "총 N개 파일 · 번호 범위 03501~03620"
  - 누락 없음: 초록색 "✅ 번호가 모두 연속되어 있습니다" 카드
  - 누락 있음: 주황/빨강 카드로 "⚠️ 누락 N건" + 구간 목록(`03532`, `03540~03542`, `03599`
    형식 — 1개짜리 구간은 단일 번호로, 2개 이상은 `시작~끝`으로 표시)
  - 인식 불가 파일이 있으면 별도 회색 카드로 파일명 목록 표시 (검사에서는 제외했다는 안내 포함)
  - "← 다시 선택" 버튼으로 idle로 복귀

## 에러 처리

- 폴더 접근 실패/취소: 기존 페이지의 `pickDir` 패턴과 동일하게 조용히 무시하고 idle 유지.
- 파일이 하나도 없는 폴더: "검사할 파일이 없습니다" 안내.
- 인식된 숫자가 하나도 없는 폴더(전부 비표준 파일명): "번호를 인식할 수 있는 파일이 없습니다"
  안내, 인식 불가 목록에 전체 파일명 표시.

## 테스트 / 검증 계획

- 타입체크
- `checkFileSequence` 순수 함수는 인자만으로 테스트 가능 — 연속/누락/중복번호/빈 배열 케이스를
  직접 호출해 결과 확인
- 브라우저: 실제 폴더(RAW+JPG 혼합, 의도적으로 몇 개 파일을 삭제/이름 바꾼 폴더)로 탭 진입 →
  스캔 → 결과 화면에서 예상한 누락 구간이 정확히 나오는지 확인
