# 전체 탭 중앙 정렬 디자인

## 목적

기능 페이지의 하위 탭 버튼을 일관되게 중앙 정렬한다. 헤더 바로 아래의 페이지 이동 탭과 본문 카드 안의 작업 탭을 모두 포함하되, 각 탭이 속한 정보 구조와 전환 기능은 유지한다.

## 범위

- 공통 `.pc-tabs`와 `.pc-tab`을 사용하는 탭 바를 중앙 정렬한다.
- 콘티 스튜디오, 사진 작업실, 고객 허브처럼 전용 링크 탭을 사용하는 상단 내비게이션을 중앙 정렬한다.
- 색감 체크, 사진 보정, 브랜드 분석, PER 병원 상세, 콘티 편집처럼 인라인 스타일로 구성된 본문 내부 탭을 중앙 정렬한다.
- 탭의 문구, 아이콘, 순서, 활성 상태, 클릭 동작, URL 이동 방식은 변경하지 않는다.
- 탭이 아닌 일반 버튼 그룹, 필터, 정렬 버튼, 단계 표시기는 변경하지 않는다.

## 정렬 기준

### 상단 페이지 이동 탭

- 헤더 바로 아래에 붙는 탭 바는 브라우저 화면 기준으로 가운데 정렬한다.
- 전역 기능 사이드바가 표시되는 데스크톱에서는 콘텐츠의 `64px` 왼쪽 여백을 보정해 실제 화면 중앙과 탭 중심을 맞춘다.
- 탭 바의 배경, sticky 위치, 블러, 그림자와 구분선은 유지한다.

### 본문 내부 탭

- 본문 카드나 작업 패널 안의 탭은 해당 부모 영역의 가로 중앙에 배치한다.
- 탭을 카드 밖이나 별도 상단 영역으로 이동시키지 않는다.
- 기존 카드 폭과 콘텐츠 흐름을 유지한다.

## 구현 방식

- 공통 탭은 CSS의 중앙 정렬 규칙으로 한 번에 처리한다.
- 콘티 스튜디오, 사진 작업실, 고객 허브의 전용 내비게이션에는 공통 식별 클래스를 부여해 같은 규칙을 재사용한다.
- 페이지 내부의 인라인 탭 그룹은 탭 컨테이너의 `justifyContent` 또는 정렬 스타일만 최소 변경한다.
- 탭 전환 상태와 이벤트 핸들러는 수정하지 않는다.
- 새 React 상태나 전역 이벤트 리스너를 추가하지 않는다.

## 반응형

- 탭 전체 폭이 가용 영역보다 작으면 데스크톱과 모바일 모두 가운데 정렬한다.
- 탭 전체 폭이 가용 영역보다 크면 가로 스크롤을 유지한다.
- 오버플로가 발생할 때 첫 번째와 마지막 탭에 모두 접근할 수 있어야 한다.
- 탭은 줄바꿈하지 않고 한 줄을 유지한다.
- 모바일의 기존 터치 스크롤과 탭 최소 클릭 영역은 유지한다.

## 접근성

- 키보드 포커스 순서는 기존 탭 순서와 동일하게 유지한다.
- 활성 탭의 시각적 구분은 기존 스타일을 유지한다.
- 링크 탭과 버튼 탭의 의미를 바꾸지 않는다.
- 가로 스크롤이 필요한 경우 키보드와 터치 입력으로 모든 탭에 접근할 수 있어야 한다.

## 검증 기준

1. 사진콘티·영상콘티 탭이 화면 중앙에 위치한다.
2. 사진 작업실과 고객 허브의 상단 탭이 화면 중앙에 위치한다.
3. 메일링, 홍보 콘텐츠, 고객 포털 등 공통 탭 바가 중앙 정렬된다.
4. 색감 체크, 사진 보정, 브랜드 분석, PER 상세, 콘티 편집 내부 탭이 각 부모 영역의 중앙에 위치한다.
5. 사이드바가 표시되어도 상단 탭의 중심이 화면 중앙과 일치한다.
6. 모바일에서 짧은 탭 그룹은 중앙 정렬되고 긴 탭 그룹은 가로 스크롤할 수 있다.
7. 탭 전환, 활성 표시, 링크 이동이 기존과 동일하게 동작한다.
8. 타입 검사와 프로덕션 빌드가 통과한다.

## 제외 사항

- 탭의 색상, 모서리, 아이콘, 타이포그래피 재디자인은 포함하지 않는다.
- 탭 순서 변경이나 그룹 통합은 포함하지 않는다.
- 일반 필터와 액션 버튼의 정렬은 변경하지 않는다.
