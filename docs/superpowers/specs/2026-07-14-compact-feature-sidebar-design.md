# 컴팩트 기능 사이드바 디자인

## 목적

기능 페이지마다 표시되는 전역 기능 사이드바가 본문 영역을 과도하게 차지하지 않도록 기본 폭을 줄인다. 빠른 기능 이동이라는 기존 목적은 유지하면서, 포토클리닉의 딥그린·오렌지·리퀴드 글래스 UI와 자연스럽게 통일한다.

## 범위

- `GlobalFeatureSidebar`의 데스크톱 표시 방식과 공통 스타일을 변경한다.
- 대시보드(`/`)와 외부 공유 세션에서 사이드바를 숨기는 기존 조건은 유지한다.
- 메뉴 데이터와 순서는 `lib/toolNav.ts`의 `ALL_TOOLS`를 그대로 사용한다.
- 개별 기능 페이지의 내부 레이아웃이나 메뉴 구성은 변경하지 않는다.
- 모바일에서는 현재처럼 전역 사이드바를 숨긴다.

## 인터랙션

### 접힌 상태

- 데스크톱 기본 폭은 `64px`다.
- 대시보드 및 각 기능의 아이콘만 중앙 정렬해 표시한다.
- 페이지 본문은 사이드바가 표시될 때 항상 왼쪽 `64px` 여백만 확보한다.
- 현재 페이지 메뉴는 딥그린 계열의 활성 배경과 작은 오렌지 포인트로 구분한다.
- 아이콘만 보이는 상태에서도 각 링크는 접근 가능한 이름을 유지하고, 포인터 사용자를 위한 메뉴명 툴팁을 제공한다.

### 확장 상태

- 사이드바 안에 마우스가 들어오거나 링크에 키보드 포커스가 들어오면 폭이 `216px`로 확장된다.
- 확장 영역은 본문 위에 겹쳐 표시한다. 본문의 왼쪽 여백은 `64px`로 유지해 레이아웃 이동을 방지한다.
- 확장 시 메뉴명이 나타나며, 텍스트는 한 줄로 유지한다.
- 마우스가 벗어나고 내부 포커스도 없으면 다시 `64px`로 접힌다.
- 폭과 메뉴명 노출은 짧고 부드러운 전환을 사용한다. `prefers-reduced-motion` 사용자는 전환 시간을 최소화한다.

## 시각 디자인

- 배경은 공통 `--liquid-panel-strong`, 테두리는 `--liquid-border`, 그림자는 `--liquid-shadow-strong`을 활용한다.
- 배경 블러와 채도를 적용해 기존 헤더·카드의 리퀴드 글래스 질감을 이어간다.
- 기본 아이콘과 텍스트는 `--muted-green`, 활성 상태는 `--deep-green`, 포인트는 `--orange`를 사용한다.
- 각 메뉴의 클릭 영역은 접힌 상태에서도 최소 40px 이상을 확보한다.
- 모서리, 간격, 구분선은 기존 `.pc-*` 공통 UI보다 과장되지 않게 작고 단정하게 맞춘다.
- 스크롤바는 기능 탐색을 방해하지 않도록 얇고 낮은 대비로 표시한다.

## 컴포넌트 구조

- `GlobalFeatureSidebar`는 현재 경로와 공유 세션 여부를 판단하는 기존 책임을 유지한다.
- 접힘과 펼침은 CSS의 `:hover`와 `:focus-within`으로 처리한다. 별도의 React 상태나 전역 이벤트 리스너를 추가하지 않는다.
- 메뉴 링크에는 접힌 상태에서 이해할 수 있는 접근성 속성과 툴팁용 텍스트를 제공한다.
- 본문 여백은 컴포넌트와 CSS가 동일한 `64px` 기준을 사용하도록 한 곳에서 명확히 관리한다.

## 반응형 및 접근성

- `900px` 이하에서는 기존과 동일하게 사이드바와 본문 왼쪽 여백을 제거한다.
- 키보드 사용자는 Tab으로 메뉴에 진입하는 즉시 확장된 메뉴명을 확인할 수 있다.
- 활성 링크에는 시각 강조와 함께 `aria-current="page"`를 제공한다.
- 링크의 포커스 윤곽은 브랜드 컬러로 명확하게 표시한다.
- 아이콘만 표시되는 상태에서도 스크린 리더가 메뉴명을 읽을 수 있어야 한다.

## 검증 기준

1. 대시보드에서는 사이드바가 표시되지 않는다.
2. 일반 기능 페이지에서 기본 폭과 본문 여백이 `64px`다.
3. hover 및 focus-within 시 사이드바만 `216px`로 확장되고 본문 위치는 변하지 않는다.
4. 현재 경로에 해당하는 메뉴가 일관되게 강조된다.
5. 긴 기능명도 확장 상태에서 잘리거나 두 줄이 되지 않는다.
6. 사이드바 내부 스크롤로 모든 기능에 접근할 수 있다.
7. 모바일과 외부 공유 세션에서는 사이드바가 숨겨진다.
8. 키보드 탐색, 포커스 표시, 접근 가능한 메뉴명이 정상 동작한다.
9. reduced-motion 환경에서 불필요한 애니메이션이 최소화된다.
10. 타입 검사와 프로덕션 빌드가 통과한다.

## 제외 사항

- 기능 메뉴의 그룹 재구성, 검색, 즐겨찾기, 사용자별 정렬은 이번 작업에 포함하지 않는다.
- 개별 기능 페이지의 헤더 또는 내부 내비게이션은 변경하지 않는다.
- 모바일용 별도 드로어 메뉴는 추가하지 않는다.
