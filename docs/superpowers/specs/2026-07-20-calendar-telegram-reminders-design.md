# 캘린더 텔레그램 알람 설계

## 목표

캘린더 일정별로 알람을 켜거나 끄고, 알람이 켜진 일정은 지정 시점에 올리비아가 텔레그램 메시지로 알려준다.

## 사용자 경험

- 시작 시간이 있는 일정만 알람을 켤 수 있다.
- 알람 시점은 정시, 10분 전, 30분 전, 1시간 전, 하루 전을 지원한다.
- 새 일정의 기본 알람 시점은 30분 전이며 기본 상태는 꺼짐이다.
- 편집 화면에서도 알람 상태와 시점을 변경할 수 있다.
- 일정 카드에는 알람이 켜진 상태를 종 아이콘으로 표시한다.
- 완료, 삭제, 알람 해제된 일정에는 메시지를 보내지 않는다.

## 데이터와 발송

`calendar_tasks`에 알람 활성화, 사전 알림 분, 발송 예정 시각, 선점 시각, 발송 시각, 시도 횟수, 최근 오류를 저장한다. 일정 날짜와 시간은 Asia/Seoul 기준으로 해석해 UTC `timestamptz` 발송 시각을 계산한다.

매분 실행되는 크론은 Postgres 함수의 `FOR UPDATE SKIP LOCKED`로 발송 대상을 원자적으로 선점한다. 성공 시 발송 시각을 기록하고, 실패 시 선점을 해제해 최대 3회까지 다시 시도한다. 이 구조로 크론이 겹쳐 실행되어도 같은 알람이 중복 발송되지 않게 한다.

메시지는 `TELEGRAM_NOTIFICATION_CHAT_ID`를 우선 사용하고, 없으면 기존 `TELEGRAM_ALLOWED_USER_ID`를 사용한다. 제목은 `🔔 올리비아 일정 알림`이며 일정명, 날짜, 시간, 장소, 메모를 포함한다.

## 보안과 검증

- 크론은 기존 `CRON_SECRET` Bearer 인증을 사용한다.
- 텔레그램 토큰과 채팅 ID는 서버 환경변수에서만 읽는다.
- 사전 알림 분은 허용된 값만 저장한다.
- 타입 검사, 테스트, 프로덕션 빌드와 실제 캘린더 폼 화면을 검증한다.
