# 프롬프터 문단 가이드라인 중앙 정렬 설계

## 목표

프롬프터 실행화면에서 `이전 문단` 또는 `다음 문단`을 누르면 선택한 문단의 실제 텍스트 영역 세로 중앙을 현재 가이드라인 위치에 맞춘다. 세 줄 문단은 두 번째 줄, 다섯 줄 문단은 세 번째 줄의 중심이 가이드라인과 겹쳐야 한다.

## 동작

- 문단 이동 시 자동 재생을 정지한다.
- 화자명과 문단 간격은 중앙 계산에서 제외하고 실제 문장 `<p>` 영역만 측정한다.
- 텍스트 영역의 `getBoundingClientRect()` 중앙과 가이드라인의 화면 좌표 차이를 계산한다.
- 일반 화면에서는 좌표 차이만큼, 상하반전 화면에서는 반대 방향으로 스크롤한다.
- 스크롤 범위를 안전하게 제한하고 진행 게이지를 즉시 동기화한다.
- 가이드라인이 꺼져 있어도 저장된 가이드 위치를 정렬 기준으로 사용한다.
- 첫 문단과 마지막 문단도 기존 상하 여백 안에서 중앙 정렬한다.

## 구현 범위

- `PrompterClient`에 문단 텍스트 요소 참조 배열을 추가한다.
- 기존 문단 래퍼 참조는 강조 표시와 마지막 문단 감지에 그대로 사용한다.
- `scrollToParagraph`만 텍스트 중앙 정렬 방식으로 변경한다.
- 리모컨의 이전/다음 문단 명령도 같은 함수를 사용하므로 동일하게 동작한다.

## 검증

- 홀수 줄 문단 1·3·5줄의 중앙 정렬 계산을 확인한다.
- 가이드 위치 변경과 상하반전 상태를 확인한다.
- 타입 검사와 프로덕션 빌드를 실행한다.
- Git 커밋, 푸시, Vercel 배포는 수행하지 않는다.
