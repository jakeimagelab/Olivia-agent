# Olivia UI 통합 및 견적 추가 옵션 디자인 명세

작성일: 2026-07-18

## 1. 목표

Olivia의 공통 헤더, 세부 탭, 강조 색상과 주요 카드 배치를 일관된 체계로 통합한다. 기능 로직은 유지하면서 화면마다 중복되거나 헤더 안에 섞여 있는 조작 요소를 콘텐츠 영역으로 이동한다. 견적서에는 새로운 복합 인원 추가 옵션을 최우선으로 반영한다.

## 2. 색상 체계

- 기본 청록: `#103A62` — 기존 UI 포인트 블루를 대체한다.
- 실행 오렌지: `#E85D2C` — 주요 실행 버튼, 현재 단계, 주의가 필요한 액션에 사용한다.
- 카드 아이보리: `#FBF7EF` — 개별 기능 홈 카드 배경에 사용한다.
- 달력 토요일, 그림 도구 팔레트, 분석 결과처럼 색 자체가 의미를 가지는 블루는 유지한다.
- 색상은 공통 CSS 변수로 관리하고 페이지별 임의 색상 사용을 줄인다.

## 3. 견적서 추가 옵션 — 최우선 구현

`견적서 > 추가 옵션`의 표시 순서는 다음과 같다.

1. 프로필 인원 추가 — 1인당 250,000원
2. 연출 인원 추가 — 1인당 450,000원
3. 프로필/연출 추가 — 1인당 650,000원
4. 인테리어 층수 추가 및 기존 나머지 옵션

새 `프로필/연출 추가` 항목은 독립 수량 입력값을 가진다. 선택 수량 × 650,000원을 옵션 합계에 더하며 견적 미리보기, 저장 데이터, 계약서 전달 데이터에 `프로필/연출 추가`라는 이름과 수량이 표시된다. 새 견적 초기화 시 수량은 0으로 돌아간다. 기존 두 인원 추가 옵션의 계산은 변경하지 않는다.

## 4. 공통 헤더와 세부 내비게이션

- 모든 일반 기능 페이지 헤더에는 포토클리닉 PNG 로고와 페이지 제목만 둔다.
- 로고 주변의 색상 배경이나 별도 도형은 제거하고 PNG 원본만 노출한다.
- 페이지 조작 버튼, 뷰 전환, 기간 선택, 불러오기 기능은 헤더 아래의 공통 액션/탭 영역으로 이동한다.
- 세부 탭은 제공된 ‘리얼 이미지 디렉터’ 참고 이미지처럼 밝은 배경의 가로 탭 바로 통일한다.
- 활성 탭은 오렌지 또는 오렌지 포인트로 구분하고 나머지 정보 구조는 `#103A62`를 사용한다.
- 모바일에서는 가로 스크롤을 허용하며 버튼의 터치 영역을 유지한다.

적용 대상에는 캘린더, 메모, 콘티, 사진 작업실, 아이디어 제안 및 공통 `PageHeader`를 사용하는 기능 페이지가 포함된다.

## 5. 홈 화면

- 홈 로고의 배경 컬러를 제거하고 PNG만 표시한다.
- 오늘의 명언 카드는 오늘 스케줄 카드와 같은 외곽선, 라운드, 배경, 여백 체계를 사용한다.
- 명언 문구와 인물명은 카드 중앙에 배치한다.
- 인물 초상은 저해상도 픽셀 블록으로 구성한 픽셀아트 스타일 아바타로 표시한다. 이미지가 없어도 레이아웃이 흔들리지 않도록 고정 비율 영역을 사용한다.

## 6. 기능 페이지 공통 배치

- 고객관리 미연결 메시지는 우측 하단 팝업이 아니라 페이지 콘텐츠 최상단의 가로 배너로 표시한다.
- 연결된 고객 정보도 같은 위치와 동일한 배너 구조를 사용한다.
- 콘티 생성의 `이전 콘티 불러오기`, `PDF / 이미지 불러오기` 버튼은 오렌지 액션 스타일로 변경한다.
- 사진 작업실의 사진 분류 페이지에 중복 노출되는 내부 헤더는 삭제한다.
- 아이디어 제안 페이지의 헤더 버튼은 헤더 아래 액션 영역으로 이동한다.
- 리뷰 콘텐츠의 수동 등록 박스는 사용 가능한 콘텐츠 너비의 중앙에 배치한다.
- 개별 기능 홈의 기능 카드는 옅은 아이보리 배경을 사용한다.

## 7. 고객관리 상세

- 상단의 짙은 청록 전체 폭 헤더는 제거하고 콘텐츠 안의 오렌지 포인트 요약 카드로 바꾼다.
- 목록 이동, 고객명, 진료과, 고객 화면 미리보기, 진행 상태는 작은 요약 카드 안에서 정돈한다.
- `NEXT ACTION` 카드는 현재보다 높이와 상하 여백을 줄이고 핵심 문구·상태·실행 버튼을 압축한다.
- 기존 블루 테두리, 진행률, 버튼은 오렌지로 변경한다.
- 단계 정보와 자동 처리 상태 등 기능 정보는 삭제하지 않는다.

## 8. 컴포넌트 경계

- `PageHeader`: 로고와 제목만 담당한다.
- 공통 탭/액션 바: 페이지별 버튼과 탭을 헤더 아래에서 표시한다.
- `ClientContextBanner`: 연결 및 미연결 상태를 콘텐츠 상단에서 표시한다.
- 페이지별 컴포넌트: 실제 기능 상태와 이벤트 처리만 담당한다.
- 색상 토큰: 청록, 오렌지, 아이보리와 파생 배경색을 한곳에서 관리한다.

## 9. 동작 및 오류 처리

- 버튼을 헤더 아래로 이동해도 기존 이벤트 핸들러와 상태는 그대로 연결한다.
- 고객 컨텍스트 조회 중, 오류, 미연결, 연결됨 상태를 모두 유지한다.
- 견적 복합 옵션의 수량은 음수가 될 수 없으며 0이면 출력 항목에서 제외한다.
- 페이지별 데이터 요청과 저장 API 규격은 변경하지 않는다.

## 10. 검증 기준

- 프로덕션 빌드와 타입 검사가 성공한다.
- 데스크톱과 모바일에서 헤더에는 로고와 제목 외 요소가 없다.
- 캘린더, 메모, 콘티, 사진 작업, 아이디어 제안의 조작 요소가 헤더 아래에 있다.
- 고객 연결 배너가 콘텐츠 상단에 표시된다.
- 고객 상세의 상단 카드와 NEXT ACTION 카드가 축소된 오렌지 체계로 보인다.
- 새 견적 옵션 1인을 선택하면 합계가 650,000원 증가하고 저장·미리보기·계약 데이터에 표시된다.
- 의미가 있는 파란색은 유지되고 기존 UI 강조용 블루는 `#103A62`로 변경된다.

## 11. 범위 밖

- 견적·CRM·사진 분류의 업무 로직 재설계
- 데이터베이스 스키마 변경
- 달력 토요일 및 그림 팔레트처럼 의미를 가진 색상의 변경
- 인물의 사실적인 초상 이미지 생성이나 외부 이미지 API 연동
