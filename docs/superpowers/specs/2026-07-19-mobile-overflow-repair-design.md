# Olivia 모바일 화면 넘침 개선 설계

## 목표

모바일 Safari/Chrome에서 AI 추천 병원 역분석, 홍보 콘텐츠 제작, 통합 메일링, 관리자 홈이 화면 너비를 넘어 잘리는 문제를 해결한다. 데스크톱 레이아웃과 기능 동작은 유지한다.

## 공통 원칙

- 모바일 작업영역과 카드의 `min-width`를 0으로 제한하고 입력 요소의 최대 너비를 부모 너비로 제한한다.
- 본문은 화면 안에서 1열로 배치한다.
- 탭과 긴 워크플로우 단계만 의도적인 가로 스크롤을 허용한다.
- 브라우저 전체에 가로 스크롤이 생기지 않게 하되, 콘텐츠를 임의 배율로 축소하지 않는다.
- 모바일 좌우 여백은 10~14px, 카드 내부 여백은 14~16px 범위로 통일한다.

## 페이지별 변경

### AI 추천 병원 역분석

- 메인/보조 패널의 2열 구조를 1열로 전환한다.
- 프로젝트 입력 폼을 1열로 전환하고 모든 입력창을 화면 안에 맞춘다.
- 분석 단계는 카드 크기를 유지한 채 해당 단계 영역 안에서만 가로 스크롤한다.
- 보조 패널의 sticky 동작은 모바일에서 해제한다.

### 홍보 콘텐츠 제작

- 콘텐츠 본문의 중첩 좌우 패딩을 제거한다.
- 블로그 4단계 버튼은 한 화면 안에 동일 폭으로 배치하고 작은 모바일 글자 크기를 사용한다.
- 스타일 카드, 상세 패널, 글 작성 폼을 1열로 전환한다.
- 채널 탭은 기존처럼 가로 스크롤하되 페이지 폭을 확장하지 않는다.

### 통합 메일링

- 입력/미리보기 2열 구조를 1열로 전환한다.
- 받는 사람, 제목, 본문 카드와 입력창의 `min-width`를 제거한다.
- 연락처 검색 버튼이 붙은 입력 행은 이메일 입력칸이 남은 폭 안에서 줄어들도록 한다.
- 최상위 콘텐츠 패딩을 모바일 기준으로 줄인다.

### 관리자 홈

- 헤더 제목에 모바일 전용 크기를 우선 적용해 전역 `h1` 규칙의 간섭을 차단한다.
- 로고, 전체 제목, 검색, 알림, 추가 버튼이 한 줄에서 겹치지 않도록 제목 영역을 유동 폭으로 만든다.
- 대시보드 카드의 기존 1열 모바일 구성은 유지한다.

## 검증

- 390px 및 430px 뷰포트에서 본문 전체 가로 스크롤이 없어야 한다.
- 탭과 워크플로우 단계 영역만 독립적으로 가로 스크롤되어야 한다.
- 입력창, 카드, 버튼이 오른쪽에서 잘리지 않아야 한다.
- TypeScript 검사, 기존 테스트, Next.js 프로덕션 빌드를 통과해야 한다.
