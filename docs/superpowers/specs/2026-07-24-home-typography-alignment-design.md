# 홈 화면 타이포 정합성 수정

## 목적

홈 화면의 빠른 실행, 업무 브리핑, 마케팅 브리핑이 Olivia의 기존 화면과 같은
NanumSquare 계열과 시각적 밀도를 사용하도록 정리한다.

## 범위

- `app/admin/admin.css`의 `.home-*` 타이포 규칙만 수정한다.
- 상단 브리핑 카드, 데이터 처리, 링크, 액션 동작은 변경하지 않는다.
- 새 폰트 파일이나 UI 라이브러리를 추가하지 않는다.

## 설계

1. `font: <weight> <size>/<line-height> inherit` 축약 선언을 제거한다.
2. 홈 대시보드 루트에 `font-family: var(--font-sans)`를 명시한다.
3. 현재 실제로 로드되는 NanumSquare Regular를 기준으로 굵기를 `400`과 `700`으로
   정규화해 브라우저의 과도한 합성 굵기를 줄인다.
4. 6.5~8.5px로 축소된 보조 텍스트를 9~11px 범위로 올리고, 제목과 본문의 위계를
   기존 관리자 UI와 맞춘다.
5. 모바일에서는 데스크톱과 같은 위계를 유지하면서 공간에 맞춰 빠른 실행의
   보조 문구만 숨긴다.

## 검증

- TypeScript 검사를 실행한다.
- 변경 diff에서 홈 전용 CSS 외 기능 코드가 바뀌지 않았는지 확인한다.
- 가능한 경우 브라우저에서 빠른 실행과 두 브리핑 패널의 실제 폰트 및 오버플로를
  확인한다.
