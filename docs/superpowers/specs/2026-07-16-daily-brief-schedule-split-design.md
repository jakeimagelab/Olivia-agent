# 관리자 홈 데일리 브리프 50:50 체크리스트 디자인

## 목표

관리자 홈 상단의 긴 데일리 브리프를 하나의 컴팩트한 카드 안에서 두 영역으로 나눈다. 왼쪽은 현재 브리프를 유지하고, 오른쪽은 오늘 일정을 직접 완료 처리할 수 있는 체크리스트로 구성한다.

## 레이아웃

- 데스크톱에서 짙은 초록색 통합 카드 내부를 50:50으로 나눈다.
- 왼쪽에는 날짜, 현재 시각, 인사말, 대기 건수, 다음 일정을 표시한다.
- 오른쪽에는 `오늘 스케줄` 제목, 완료 수, 진행률, 오늘 일정 체크리스트를 표시한다.
- 두 영역 사이에는 낮은 대비의 세로 구분선을 둔다.
- 기존 상단 `오늘의 명언` 카드는 제거한다.
- 모바일에서는 왼쪽 브리프와 오른쪽 체크리스트를 위아래로 쌓는다.
- 전체 카드 높이는 데스크톱에서 컴팩트하게 유지하며, 일정이 많으면 체크리스트 내부에서만 스크롤한다.

## 데이터와 상호작용

- 기존 `/api/dashboard` 응답의 `todayTasks`를 양쪽 영역에서 공유한다.
- 체크리스트는 미완료 항목을 먼저 배치하고, 같은 상태 안에서는 시작 시간순으로 정렬한다.
- 체크박스를 누르면 화면 상태를 즉시 변경하는 낙관적 업데이트를 적용한다.
- `PATCH /api/calendar`에 `{ id, completed }`를 전송해 캘린더 데이터에 저장한다.
- 저장 중인 항목은 중복 클릭을 막는다.
- 저장 실패 시 해당 항목을 이전 상태로 되돌리고 체크리스트 안에 오류 메시지와 재시도 안내를 표시한다.
- 완료 수, 남은 할 일 수, 진행률, 다음 일정은 체크 결과에 맞춰 즉시 다시 계산한다.

## 상태 처리

- 로딩: 좌우 카드 골격을 유지하면서 체크리스트 로딩 문구를 표시한다.
- 빈 상태: `오늘 등록된 일정이 없습니다`와 캘린더 이동 링크를 표시한다.
- 오류: 운영 데이터 연결 오류와 다시 불러오기 버튼을 표시한다.
- 정상: 체크리스트와 완료 진행률을 표시한다.

## 접근성

- 각 일정은 실제 `checkbox` 입력 또는 동등한 체크박스 역할을 사용한다.
- 저장 중 상태와 오류 문구는 보조 기술에 전달한다.
- 키보드로 체크 완료/해제가 가능해야 한다.
- 완료 항목은 취소선뿐 아니라 체크 상태와 텍스트로도 구분한다.

## 변경 범위

- `TodayAlertBanner`를 통합 브리프/체크리스트 컴포넌트로 확장한다.
- 관리자 홈에서 `DailyQuoteWidget` 렌더링을 제거한다.
- 관리자 CSS에 50:50 레이아웃, 체크리스트, 반응형 스타일을 추가한다.
- 캘린더 API와 데이터베이스 스키마는 변경하지 않는다.

## 완료 기준

1. 상단 브리프가 데스크톱에서 정확히 두 열로 보인다.
2. 오른쪽에서 오늘 일정을 직접 완료/해제할 수 있다.
3. 변경 사항이 캘린더 API에 저장된다.
4. 실패하면 UI가 이전 상태로 복구되고 오류가 표시된다.
5. 모바일에서는 두 영역이 세로로 자연스럽게 쌓인다.
6. 기존 관리자 홈의 나머지 요약 카드와 업무 영역은 변경되지 않는다.

## 검증

- TypeScript 검사와 프로덕션 빌드
- 일정 완료/해제 성공 및 실패 복구 확인
- 로딩, 빈 상태, 오류 상태 확인
- 데스크톱과 모바일 레이아웃 확인
