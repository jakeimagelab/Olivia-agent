# Olivia 고객 신규등록 자동 실행 설계

작성일: 2026-07-19

## 문제

올리비아는 고객 신규등록 요청을 `create_feature_record` 도구 호출로 준비하지만, 이 도구가 채팅의 자동 실행 목록에 포함되어 있지 않다. 사용자가 채팅 안의 `실행` 버튼을 추가로 누르지 않으면 실제 `clients` INSERT가 발생하지 않는다. 준비 메시지와 승인 카드가 등록 완료처럼 인식될 수 있어, 사용자는 고객 페이지에서 신규 고객을 찾지 못한다.

운영 `GET /api/clients`를 읽기 전용으로 확인한 결과 기존 고객은 정상 반환됐지만 방금 요청한 신규 고객 행은 존재하지 않았다. 따라서 이번 문제는 목록 캐시나 RLS가 아니라 도구가 실행 단계까지 진행되지 않은 것이다.

## 확정 동작

### 신규 고객 생성

- 사용자가 병원명 등 필수정보와 함께 고객 등록을 명시적으로 요청하면 그 요청을 생성 승인으로 간주한다.
- `create_feature_record` 중 `domain: "client"`만 승인 카드 없이 즉시 실행한다.
- INSERT 결과에서 생성된 고객 ID와 행을 반드시 확인한 뒤에만 등록 완료 메시지를 표시한다.
- 성공하면 기존 결과 계약에 따라 `/clients?id={recordId}`로 이동한다.
- 실패하면 등록 완료 문구를 표시하지 않고 서버 오류를 사용자에게 보여준다.

### 다른 데이터 생성과 기존 데이터 수정

- `create_feature_record`의 다른 domain은 기존 승인 카드를 유지한다.
- 모든 `update_feature_record`는 기존 승인 카드를 유지한다.
- 가격, 계약, 메일 발송 등 기존 owner/review 권한 정책은 변경하지 않는다.

## 구현 경계

도구 이름 전체를 자동 실행 목록에 추가하면 다른 도메인의 생성까지 자동 실행되므로 사용하지 않는다. 채팅이 tool request를 받았을 때 다음 조건을 검사한다.

```ts
tool.name === "create_feature_record" && tool.input?.domain === "client"
```

기존 자동 실행 도구 집합과 위 조건 중 하나가 참일 때만 즉시 실행한다. 같은 조건을 현재 사용 중인 채팅 컴포넌트와 레거시 호환 컴포넌트에 동일하게 적용한다.

## 데이터 흐름

1. 사용자가 신규 고객 등록을 요청한다.
2. AI가 `create_feature_record(domain: client)`를 반환한다.
3. 채팅이 고객 생성 요청임을 확인하고 즉시 `/api/olivia`에 `pendingTool`을 전송한다.
4. 서버가 `clients`에 INSERT하고 `.select("*").single()`로 생성 행을 확인한다.
5. 서버가 `recordId`와 고객 상세 URL을 반환한다.
6. 채팅이 `olivia-data-changed` 이벤트를 발생시키고 고객 상세 페이지로 이동한다.
7. INSERT 또는 검증이 실패하면 오류 메시지만 표시한다.

## 테스트

- 고객 생성 tool request만 자동 실행 대상으로 판정된다.
- 고객 수정은 자동 실행 대상이 아니다.
- 견적·계약 등 다른 domain 생성은 자동 실행 대상이 아니다.
- 고객 생성 결과는 고객 상세 URL을 반환한다.
- 전체 단위 테스트, TypeScript 검사, 프로덕션 빌드가 통과한다.

## 완료 기준

- 사용자는 명시적인 고객 등록 요청 후 별도의 실행 버튼을 누르지 않는다.
- 성공 메시지가 표시된 고객은 운영 `clients` 테이블과 고객 목록 API에 실제 존재한다.
- 생성 성공 후 해당 고객 상세 화면이 열린다.
- 기존 고객 수정과 다른 중요 데이터 생성의 승인 흐름은 바뀌지 않는다.
