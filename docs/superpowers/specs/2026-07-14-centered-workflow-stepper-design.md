# Centered Workflow Stepper Design

## Goal

가로형 워크플로우 단계 표시가 축소형 기능 네비게이터와 겹치지 않게 하고, 네비게이터를 제외한 실제 본문 영역을 기준으로 중앙에 배치한다.

## Scope

- 사진 분류, 영상 분류, 4K→FHD 변환, RAW 셀렉트처럼 페이지 상단에 놓인 가로형 단계 표시
- 영상 콘티처럼 본문 안에 놓인 가로형 단계 표시
- AI Trust Gap, 웹사이트 빌더 등 동일한 구조로 확인되는 가로형 단계 표시
- 진행률 막대, 수량 스테퍼, 세로형 워크플로우 목록, 일반 하위 탭은 제외한다.

## Layout Rules

1. 단계 표시의 기준 영역은 전체 브라우저 폭이 아니라 `.pc-fs-content` 본문 영역이다.
2. 데스크톱에서는 왼쪽 64px 기능 네비게이터를 침범하지 않으며, 본문 영역 안에서 단계 묶음을 중앙 정렬한다.
3. 단계 묶음이 본문보다 넓으면 첫 단계가 잘리지 않도록 안전한 가로 스크롤을 제공한다.
4. 모바일에서는 단계가 화면에 들어오면 중앙 정렬하고, 넘치면 왼쪽부터 모든 단계를 스크롤해 확인할 수 있게 한다.
5. 기존 단계명, 순서, 완료·활성·대기 상태 색상과 동작은 변경하지 않는다.

## Implementation

- `app/globals.css`에 재사용 가능한 워크플로우 바와 트랙 클래스를 추가한다.
- 바는 `overflow-x: auto`를 유지하고, 트랙은 `width: max-content`, `min-width: 100%`, `justify-content: safe center` 조합으로 중앙 정렬과 안전한 오버플로를 함께 처리한다.
- 페이지별 인라인 레이아웃은 유지하되 공통 클래스만 부여해 중복 수정을 줄인다.
- 연결선이 유동 폭을 사용하는 영상 콘티·웹사이트 빌더형 UI는 바깥 래퍼를 본문 폭에 맞추고, 내부 단계 구조와 연결선 동작은 유지한다.
- 기능 네비게이터가 없는 화면에서도 같은 클래스가 자연스럽게 컨테이너 중앙을 기준으로 동작하게 한다.

## Responsive Behavior

- 데스크톱: 네비게이터 오른쪽의 본문 영역 중앙에 표시한다.
- 좁은 화면: 트랙이 짧으면 중앙 정렬한다.
- 트랙이 길면 중앙 정렬로 첫 항목이 잘리는 문제를 피하고, 시작점부터 가로 스크롤할 수 있게 한다.
- 기존 모바일 단계 라벨 숨김 규칙은 유지한다.

## Verification

- 관련 페이지를 정적 검색해 모든 가로형 워크플로우 표시가 공통 규칙을 사용하는지 확인한다.
- 타입 검사와 프로덕션 빌드를 실행한다.
- 대표 페이지인 사진 분류를 데스크톱 크기로 열어 첫 단계가 네비게이터에 가려지지 않고 본문 중앙에 놓이는지 확인한다.
- 좁은 화면에서 첫 단계 접근성과 가로 스크롤을 확인한다.
