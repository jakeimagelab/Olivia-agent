# 고객 생성 시 기존 갤러리 역연결 설계

작성일: 2026-07-19

## 문제

`photo_galleries`는 고객보다 먼저 생성될 수 있다. 이 경우 병원명은 저장되지만 `client_id`와 `workflow_run_id`는 null이다. 고객 상세 화면은 `client_id`로 갤러리를 조회하므로 같은 병원명의 갤러리가 있어도 빈 목록으로 표시한다.

운영 DB에서 `반포리움성형외과` 갤러리는 존재하지만 두 연결 ID가 모두 null인 것을 확인했다.

## 연결 규칙

- 정규화한 병원명이 동일한 갤러리만 후보로 삼는다.
- `client_id is null`인 갤러리만 새 고객에게 연결한다.
- 이미 다른 고객에게 연결된 갤러리는 수정하지 않는다.
- 활성 워크플로우 ID가 있으면 `workflow_run_id is null`인 갤러리에 함께 기록한다.
- 같은 작업이 반복되어도 결과가 달라지지 않는 멱등 처리로 구현한다.

## 적용 시점

1. 올리비아 고객 생성 직후 기존 갤러리를 역연결한다.
2. 일반 고객 생성 API에서도 동일하게 역연결한다.
3. 고객 상세의 갤러리 조회 결과가 비어 있으면 병원명으로 미연결 갤러리를 찾아 역연결한 뒤 한 번 재조회한다.

## 조회 정책

정상 조회는 계속 `client_id` 인덱스를 사용한다. 병원명 검사는 연결이 비어 있을 때만 수행해 일상적인 상세 조회 비용을 늘리지 않는다.

## 오류 처리

- 역연결 실패가 고객 생성 자체를 롤백하지 않는다.
- 상세 갤러리 조회는 연결 보정 실패 시에도 기존 빈 결과를 반환할 수 있으며 서버 오류는 로그로 추적한다.
- 잘못 연결하는 것보다 미연결 상태를 유지하는 것을 우선한다.

## 테스트

- 공백과 대소문자가 다른 동일 병원명을 연결한다.
- 다른 병원명은 연결하지 않는다.
- 이미 client_id가 있는 갤러리는 변경 대상에서 제외한다.
- 연결 대상이 없으면 안전하게 종료한다.
- 고객 생성 및 갤러리 조회 경로가 공통 연결 함수를 사용한다.

## 완료 기준

- 반포리움 고객 상세에서 기존 촬영 갤러리가 표시된다.
- 이후 갤러리를 먼저 만들고 고객을 생성해도 자동으로 연결된다.
- 기존에 다른 고객에게 연결된 갤러리는 보존된다.
