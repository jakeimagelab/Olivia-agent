# 중앙 정렬 페이지 헤더 디자인

## 목적

모든 기능 페이지의 상단 헤더를 같은 구조로 통일한다. 포토클리닉 로고와 페이지 설명 텍스트를 화면 정중앙에 고정하고, 모든 기능 버튼을 오른쪽에 모아 시각적 위계와 사용 흐름을 단순화한다. 전역 기능 사이드바는 페이지 헤더와 같은 아주 연한 초록 배경을 사용한다.

## 범위

- 공통 `.pc-header`를 사용하는 기능 페이지의 헤더 정렬을 변경한다.
- 공통 `PageHeader`와 페이지별로 직접 작성된 헤더에 동일한 규칙을 적용한다.
- 헤더에 있는 `관리자 홈` 링크와 그에 딸린 구분선을 제거한다.
- 고객 목록, 고객 화면, 견적서, 구독 관리 등 업무 이동 버튼은 삭제하지 않고 오른쪽 기능 영역으로 이동한다.
- 저장, 다운로드, 내보내기, 상태 표시 등 기존 오른쪽 기능은 유지한다.
- 본문 또는 빈 상태 화면에 있는 `관리자 홈으로` CTA는 헤더가 아니므로 변경하지 않는다.
- `.ops-header`처럼 로고형 공통 헤더가 아닌 소개 영역에서는 `관리자 홈` 링크만 제거하고 자체 제목 구조는 유지한다.

## 헤더 구조

헤더는 세 영역으로 구성한다.

1. 왼쪽 영역은 항상 비워 둔다.
2. 가운데 영역에는 포토클리닉 로고와 현재 페이지를 설명하는 텍스트를 배치한다.
3. 오른쪽 영역에는 해당 페이지에서 사용할 수 있는 모든 버튼과 업무 이동 링크를 배치한다.

가운데 영역은 좌우 콘텐츠의 폭과 무관하게 화면 기준 정중앙에 위치해야 한다. 이를 위해 헤더는 좌우가 같은 폭을 갖는 3열 CSS Grid를 사용한다. 가운데 브랜드 영역은 중앙 열에, 기능 영역은 오른쪽 열의 끝에 정렬한다.

## 공통 컴포넌트

- `PageHeader`는 더 이상 관리자 홈 링크를 렌더링하지 않는다.
- 기존 `backHref`, `backLabel`처럼 관리자 홈 이동을 위한 속성은 제거하거나 사용처를 정리한다.
- 공통 헤더의 브랜드 영역은 로고와 `title`을 계속 표시한다.
- `actions`는 오른쪽 기능 영역에 유지한다.
- 페이지별 직접 작성 헤더는 브랜드만 중앙 영역에 남기고, 업무 이동 링크와 기능 버튼을 `.pc-header-actions` 안으로 모은다.

## 시각 디자인

- 전역 기능 사이드바 배경은 헤더와 동일한 `--liquid-bg`를 사용한다.
- 사이드바의 블러, 테두리, 그림자와 접힘·펼침 동작은 유지한다.
- 헤더 높이, 로고 크기, 페이지 설명 텍스트의 색상과 타이포그래피는 기존 디자인 시스템을 유지한다.
- 헤더 왼쪽에는 버튼, 구분선, 장식 요소를 두지 않는다.
- 오른쪽 버튼의 간격과 크기는 기존 `.pc-header-actions` 규칙을 따른다.

## 반응형

- 데스크톱에서는 로고와 설명 텍스트를 모두 화면 중앙에 표시한다.
- 모바일에서는 기존과 같이 설명 텍스트를 숨기고 로고만 화면 중앙에 표시한다.
- 오른쪽 기능 영역은 화면 폭을 넘길 경우 가로 스크롤을 허용한다.
- 기능 영역은 중앙 브랜드 영역을 침범하지 않도록 최대 폭을 제한한다.
- 왼쪽 열은 모바일에서도 비어 있어야 한다.

## 접근성

- 이동 링크를 오른쪽으로 옮겨도 링크 텍스트와 접근 가능한 이름은 유지한다.
- 기능 버튼의 포커스 순서는 오른쪽 영역의 시각 순서와 일치해야 한다.
- 로고의 대체 텍스트는 `포토클리닉`으로 유지한다.
- 헤더에서 삭제되는 관리자 홈 링크를 대신하는 전역 이동 수단은 왼쪽 기능 사이드바의 대시보드 항목이다.

## 검증 기준

1. 헤더 왼쪽에는 어떤 버튼이나 링크도 표시되지 않는다.
2. 로고와 페이지 설명 텍스트가 화면 기준 정중앙에 위치한다.
3. 저장, 다운로드, 업무 이동 링크 등 모든 헤더 기능이 오른쪽에 위치한다.
4. 헤더의 `관리자 홈` 링크와 관련 구분선이 남아 있지 않다.
5. 본문의 `관리자 홈으로` CTA는 유지된다.
6. 사이드바 배경과 헤더 배경이 동일한 아주 연한 초록 계열이다.
7. 데스크톱과 모바일에서 중앙 브랜드와 오른쪽 기능이 겹치지 않는다.
8. 외부 공유 세션에서도 헤더 구조가 깨지지 않는다.
9. 타입 검사와 프로덕션 빌드가 통과한다.

## 제외 사항

- 개별 페이지의 본문 구조와 기능 로직은 변경하지 않는다.
- 헤더 로고 이미지와 페이지 설명 문구 자체는 변경하지 않는다.
- 사이드바 메뉴 구성, 순서, 접힘·펼침 동작은 변경하지 않는다.
